## Context

当前 iOS 新闻应用采用 SwiftUI ScrollView + LazyVStack 的扁平列表布局，每个 NewsCardView 直接堆叠。用户希望引入时间线设计，将时间维度可视化，同时让卡片呈现极简悬浮风格。

现有设计系统（DESIGN.md）定义了完整的色彩、字体、间距 token，时间线设计将完全复用这些 token，不引入新色彩。

## Goals / Non-Goals

**Goals:**
- 左侧垂直时间线 + 右侧新闻卡片的双栏布局
- 时间线上的圆点与卡片关联，圆点旁显示小时:分钟
- 卡片采用无边框、微妙阴影的悬浮极简风格
- 支持无限滚动和下拉刷新
- 响应式适配不同屏幕尺寸

**Non-Goals:**
- 不改变 API 数据结构或网络层
- 不引入日期分组（日/周分隔线），保持纯时间线流
- 不添加卡片展开/折叠交互
- 不修改设计系统 token

## Decisions

### 1. 时间线布局方案：HStack 双栏

**选择**: 使用 HStack 将屏幕分为左侧时间线列和右侧卡片列。

**理由**: SwiftUI 原生 HStack 天然支持双栏对齐，配合 GeometryReader 可精确控制比例。相比 ZStack 叠加方案，HStack 的对齐更可控，不会出现时间线和卡片错位的问题。

**替代方案**: ZStack 将时间线覆盖在卡片上方 — 难以保证圆点与卡片垂直对齐，尤其在动态高度卡片场景下。

### 2. 时间线列宽度：固定 60pt

**选择**: 左侧时间线列固定 60pt 宽，包含圆点、时间文字和垂直线。

**理由**: 时间文字（如 "14:30"）在 SF Pro Text 12px 下约需 35pt，加上圆点（8pt）和间距，60pt 足够且不浪费空间。固定宽度避免动态计算带来的布局抖动。

### 3. 圆点样式：8pt 实心圆，Action Blue

**选择**: 8pt 直径实心圆，使用 `{colors.primary}` (#0066cc)。

**理由**: Action Blue 是设计系统中唯一的交互色，用在时间线圆点上既保持一致性，又能让时间线成为视觉锚点。8pt 在 12px 时间文字旁比例协调。

### 4. 卡片悬浮感实现

**选择**: 无边框、背景 `{colors.canvas}`、微妙阴影 `rgba(0, 0, 0, 0.06) 0 2px 12px`、圆角 `{rounded.lg}` (18px)、内边距 `{spacing.lg}` (24px)。

**理由**: 参考 DESIGN.md 中产品阴影的克制使用原则，卡片阴影比产品阴影更轻（0.06 alpha vs 0.22），营造"浮在时间线上"的感觉而不喧宾夺主。18px 圆角与设计系统中 store-utility-card 一致。

### 5. 响应式策略

**选择**: 屏幕宽度 < 736pt 时，隐藏时间线列，卡片变为全宽标准列表（保留悬浮卡片风格）。

**理由**: iPhone 竖屏下 60pt 时间线列会压缩卡片宽度，影响阅读体验。iPad 和横屏模式保留完整时间线布局。

## Risks / Trade-offs

- **[卡片高度变化导致圆点偏移]** → 使用 LazyVStack 的自然布局，圆点与卡片顶部对齐（alignment: .top），不要求垂直居中
- **[时间线在快速滚动时可能闪烁]** → 时间线使用独立的固定视图层，不参与 LazyVStack 的回收
- **[同分钟多条新闻]** → 允许多个卡片共享同一分钟标注，不做去重处理
